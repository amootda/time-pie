# Time Pie - Product Requirements Document (PRD)

## 1. 개요

### 1.1 제품 비전
Time Pie는 사용자의 루틴과 습관을 분석하여, **무리한 계획을 방지하고 실현 가능한 하루의 여백을 찾아주는** 시간 관리 서비스입니다. 원형 시간표를 통해 시간을 파이처럼 시각화하고, AI가 유동적 일정을 추적하여 데이터 기반으로 더 나은 시간 사용을 가이드합니다.

### 1.2 문제 정의
현재 사용자들은 일정 관리(Google Calendar), 할 일 관리(Todoist), 습관 추적(Habitica) 등 여러 앱을 따로 사용해야 합니다. 이로 인해:
- 앱 간 전환으로 인한 피로감
- 데이터가 분산되어 전체적인 시간 관리가 어려움
- 습관과 일정, 할 일의 연계가 불가능
- 무리한 계획을 세우고 반복적으로 실패하는 악순환
- 자신의 실제 시간 사용 패턴을 객관적으로 파악하기 어려움

### 1.3 솔루션
Time Pie는 일정을 타입별로 분류하고, AI가 유동적 일정을 추적·분석하여 사용자의 실현 가능한 하루를 설계할 수 있도록 돕습니다. 데이터 기반 패턴 분석으로 상습적 실패 구간을 탐지하고, 자주 관측되는 일정에 대해 조정 가이드를 제공합니다.

## 2. 타겟 사용자

### 2.1 주요 페르소나

**페르소나 1: 자기계발을 원하는 직장인 (25-35세)**
- 업무 외 시간에 운동, 독서 등 습관을 만들고 싶음
- 여러 앱을 사용하다 지쳐서 하나로 통합하고 싶음
- 간단하고 빠른 입력을 선호

**페르소나 2: 대학생/취준생 (20-28세)**
- 수업, 과제, 자격증 공부 등 다양한 일정 관리 필요
- 매일 꾸준히 해야 할 공부 습관 형성이 목표
- 모바일 사용이 주력

### 2.2 사용자 니즈
- 한눈에 오늘 해야 할 일정, 할 일, 습관을 확인
- 습관 완료 시 성취감을 느낄 수 있는 시각적 피드백
- 빠르고 간편한 입력

## 3. 핵심 기능 (MVP)

### 3.1 일정 타입 분류
| 항목 | 설명 |
|------|------|
| 기능 | 일정을 타입으로 구분하여 관리 방식을 차별화 |
| 상세 | - **고정 일정**: 출근, 수업 등 시간이 정해진 일정<br>- **유동 일정**: 공부, 책 읽기 등 시간 배분이 유동적인 일정 (AI 추적 대상)<br>- **반복 일정**: 매일/매주 반복되는 루틴성 일정<br>- 타입에 따라 파이 차트에서 시각적으로 구분 표시 |
| 우선순위 | P0 (필수) |

### 3.2 AI 기반 유동 일정 추적
| 항목 | 설명 |
|------|------|
| 기능 | 공부, 책 읽기 등 분석이 필요한 유동적 일정을 AI가 추적하여 관리 |
| 상세 | - 유동 일정의 실제 수행 시간/빈도를 자동 기록<br>- 계획 대비 실제 수행 데이터를 비교 분석<br>- 사용자의 실현 가능한 시간 배분을 AI가 제안<br>- 무리한 계획을 사전에 감지하여 경고 |
| 우선순위 | P1 (중요) |

### 3.3 데이터 기반 패턴 분석
| 항목 | 설명 |
|------|------|
| 기능 | 사용자의 시간 사용 패턴을 데이터 기반으로 추적 및 분석 |
| 상세 | - **상습 실패 탐지**: 반복적으로 달성하지 못하는 일정/습관 구간을 자동 식별<br>- **미흡 구간 탐지**: 계획 대비 수행률이 낮은 시간대를 분석<br>- **일정 조정 가이드**: 자주 관측되는 일정 패턴에 대해 현실적인 조정안 제시<br>- 주간/월간 패턴 리포트 제공 |
| 우선순위 | P1 (중요) |

### 3.4 파이 차트 뷰 (Time Pie 핵심 UI)
| 항목 | 설명 |
|------|------|
| 기능 | 24시간을 원형 파이 차트로 시각화하여 일정 표시 |
| 상세 | - **원형 시간표**: 24시간을 360도 원형으로 표현<br>- 일정이 파이 조각처럼 시간대별로 색상 구분되어 표시<br>- 일정 타입(고정/유동/반복)에 따른 시각적 구분<br>- 현재 시간을 가리키는 시계 바늘 UI<br>- 빈 시간대는 회색/투명으로 표시 (여백 시각화)<br>- 일정 조각 탭하여 상세 보기/수정<br>- 드래그로 일정 시간 조정<br>- 중앙에 오늘 날짜 및 요약 정보 표시 |
| 우선순위 | P0 (필수) |

### 3.5 캘린더 뷰
| 항목 | 설명 |
|------|------|
| 기능 | 주/월 단위로 일정 확인 및 관리 |
| 상세 | - 주간/월간 그리드 뷰<br>- 드래그 앤 드롭으로 일정 이동<br>- 일정 생성/수정/삭제<br>- 날짜 선택 시 해당 날짜의 파이 차트 뷰로 이동<br>- 투두 및 습관과 같은 화면에서 확인 가능 |

|------|------|
| 기능 | 할 일 목록 생성 및 관리 |
| 상세 | - 할 일 추가/수정/삭제<br>- 완료 체크 및 체크 해제<br>- 마감일 설정 (선택)<br>- 우선순위 설정 (높음/중간/낮음)<br>- 캘린더에서 마감일 있는 투두 표시 |
| 우선순위 | P0 (필수) |

### 3.7 습관 트래커
| 항목 | 설명 |
|------|------|
| 기능 | 반복적인 습관 생성 및 추적 |
| 상세 | - 습관 생성 (이름, 반복 주기, 목표 횟수)<br>- 일일 완료 체크<br>- 스트릭(연속 달성) 표시<br>- 주간/월간 달성률 시각화<br>- 캘린더에 습관 완료 여부 표시 |
| 우선순위 | P0 (필수) |

### 3.8 알림/리마인더
| 항목 | 설명 |
|------|------|
| 기능 | 일정, 할 일, 습관에 대한 알림 |
| 상세 | - 일정 시작 전 알림 (설정 가능)<br>- 투두 마감일 알림<br>- 습관 리마인더 (매일 특정 시간)<br>- 푸시 알림 (모바일) / 브라우저 알림 (웹) |

## 4. 사용자 플로우

### 4.1 온보딩
```
앱 시작 → 간단한 소개 (3단계) → 첫 습관 생성 유도 → 메인 화면
```

### 4.2 일일 사용 플로우
```
앱 열기 → 오늘 뷰 확인 (일정 + 투두 + 습관)
    ├── 습관 완료 체크
    ├── 투두 완료 체크
    ├── 새 일정/투두 추가
    └── 캘린더로 이동하여 주간/월간 확인
```

### 4.3 습관 완료 플로우
```
습관 항목 탭 → 완료 체크 → 스트릭 업데이트 → 축하 애니메이션(선택)
```

## 6. 기술 스택 (제안)

## 9. 성공 지표 (KPI)

### 9.1 핵심 지표
| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| DAU (일간 활성 사용자) | 출시 3개월 후 1,000명 | 일일 로그인 수 |
| 리텐션 (7일) | 40% 이상 | 7일 후 재방문 비율 |
| 습관 완료율 | 60% 이상 | 체크된 습관 / 전체 습관 |
| 평균 세션 시간 | 3분 이상 | 앱 사용 시간 |

## 12. 부록

### 12.1 경쟁 앱 분석

| 앱 | 강점 | 약점 | Time Pie 차별점 |
|----|------|------|-----------------|
| Google Calendar | 일정 관리 완벽 | 투두/습관 없음 | 통합 제공 |
| Todoist | 투두 기능 강력 | 캘린더 뷰 약함 | 캘린더 통합 |
| Habitica | 게이미피케이션 | 복잡한 UI | 심플한 UX |
| Notion | 커스터마이징 | 러닝커브 높음 | 즉시 사용 가능 |

### 12.2 용어 정의
- **스트릭 (Streak)**: 습관을 연속으로 완료한 일수
- **투두 (Todo)**: 완료해야 할 개별 작업
- **이벤트 (Event)**: 특정 시간에 예정된 일정
