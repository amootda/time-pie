# Timeblocks 캘린더 뷰 심층 분석

**작성일**: 2026-02-18
**목적**: Timeblocks 앱의 캘린더 시각화 방식을 분석하여 Time Pie에 적용

## 📊 Timeblocks 캘린더 뷰 핵심 특징

### 1. 월간 캘린더 뷰 (Primary View)

**레이아웃**:
```
┌─────────────────────────────────────────┐
│  2월 2026                    [오늘] [+] │
├─────────────────────────────────────────┤
│  일   월   화   수   목   금   토        │
├─────────────────────────────────────────┤
│  26  27  28  29  30  31   1            │
│      업무 업무 미팅                      │
│          운동     점심                   │
│                                         │
│   2   3   4   5   6   7   8            │
│  회의 휴식 개인                          │
│  운동 공부                               │
│                                         │
│  ...                                    │
└─────────────────────────────────────────┘
```

**핵심 원칙**:
- **확장 가능한 높이**: 일정이 많은 날은 셀이 늘어남 (스크롤 가능)
- **한눈에 파악**: 한 달 전체를 한 화면에 표시
- **밀도 조절**: 일정 수에 따라 자동 조정

### 2. 드래그 앤 드롭 인터랙션

**종이 다이어리 메타포**:
- 일정을 다른 날짜로 드래그하여 이동
- 포스트잇을 옮기는 것처럼 직관적
- 시각적 피드백 (들어올리기, 그림자 효과)

**사용 시나리오**:
```
1. 일정 선택 → 롱프레스 or 클릭
2. 드래그 시작 → 반투명 상태로 변경
3. 다른 날짜로 이동 → 드롭 존 하이라이트
4. 드롭 → 일정 이동 완료 + 애니메이션
```

### 3. 시각적 디자인 시스템

**색상 코딩**:
- 카테고리별 색상 (work: 파란색, personal: 초록색, etc.)
- 스티커 & 마스킹 테이프 (날짜 배경 꾸미기)
- 테마 시스템 (디자이너 협업)

**타이포그래피**:
- 날짜: 큰 숫자 (가독성 우선)
- 일정 제목: 작은 텍스트 (간결하게)
- 요일: 상단 고정 (스크롤 시 sticky)

**여백과 간격**:
- 셀 간격: 명확한 구분선
- 일정 간격: 1-2px (밀도 유지)
- 패딩: 충분한 터치 영역

### 4. 정보 밀도 vs 가독성

**Timeblocks의 선택**:
- ✅ 정보 밀도 우선: 한 화면에 최대한 많은 정보
- ✅ 확장 가능: 필요할 때만 늘어남
- ❌ 개별 일정 상세: 클릭해야 볼 수 있음

**Time Pie와 비교**:
| 특성 | Timeblocks | Time Pie (현재) |
|------|-----------|----------------|
| 뷰 범위 | 1개월 | 1일 |
| 시간 표현 | 간접적 (날짜만) | 직접적 (24시간 원형) |
| 정보 밀도 | 높음 (30일) | 낮음 (1일 상세) |
| 장기 계획 | 우수 | 부족 |
| 시간 분배 | 약함 | 우수 |

## 🎨 Timeblocks vs Time Pie 차별화 전략

### Time Pie의 강점 유지

**1. 파이 차트 (고유 가치)**:
- 24시간 시각화 → 시간 분배 직관적
- 색상으로 균형 파악 (work vs personal)
- 비어있는 시간 명확히 표시

**2. 실행 추적 (Execution Timer)**:
- Soft 이벤트 시작/완료 추적
- 실제 vs 계획 비교

### Timeblocks 벤치마킹 포인트

**1. 월간 뷰 추가 (보완적)**:
```
[Day] [Week] [Month] ← 탭

Day View: 기존 파이 차트 (메인)
Week View: 7개의 미니 파이 차트
Month View: Timeblocks 스타일 캘린더
```

**2. 드래그 앤 드롭**:
- 파이 차트에서: 조각 드래그 → 시간 조정
- 월간 뷰에서: 일정 드래그 → 날짜 이동

**3. 확장 가능한 레이아웃**:
- 월간 뷰: 일정 많은 날은 셀 높이 증가
- 주간 뷰: 스크롤 가능한 수평 레이아웃

## 🚀 Time Pie 개선 제안 (Timeblocks 벤치마킹)

### Phase 1: 주간 뷰 추가 (우선순위: 높음)

**목표**: 일주일 패턴을 한눈에 파악

**디자인**:
```
┌─────────────────────────────────────────┐
│  2월 10일 - 16일              [이번 주] │
├─────────────────────────────────────────┤
│  월    화    수    목    금    토    일  │
│  🥧    🥧    🥧    🥧    🥧    🥧    🥧  │
│  ●●●  ●●   ●●●● ●●   ●●●  ●    ●●  │
│  5개  3개  7개  3개  5개  1개  3개   │
└─────────────────────────────────────────┘

터치: 미니 파이 클릭 → 해당 날짜 Day 뷰
좌우 스와이프: 이전/다음 주
```

**구현**:
- `WeeklyPieView.tsx` 컴포넌트
- 7개의 PieChart 재사용 (size: 80px)
- 이벤트 개수 표시 (밀도 파악)

### Phase 2: 월간 뷰 추가 (우선순위: 중간)

**목표**: 장기 계획 및 패턴 파악 (Timeblocks 스타일)

**디자인 A: 이벤트 밀도 히트맵**
```
┌─────────────────────────────────────────┐
│  2월 2026                          [오늘]│
├─────────────────────────────────────────┤
│  일  월  화  수  목  금  토              │
├─────────────────────────────────────────┤
│      26  27  28  29  30  31   1         │
│      ░░  ▓▓  ░░  ▓▓  ░░  ░░  ░░        │ (밀도)
│                                         │
│   2   3   4   5   6   7   8            │
│  ▓▓  ░░  ▓▓▓ ░░  ▓▓  ░░  ░░           │
│                                         │
│  ...                                    │
└─────────────────────────────────────────┘

색상:
- ░░ (연한 회색): 0-2개 이벤트
- ▓▓ (중간 cyan): 3-5개 이벤트
- ▓▓▓ (진한 cyan): 6+ 이벤트
```

**디자인 B: Timeblocks 스타일 리스트**
```
┌─────────────────────────────────────────┐
│  2월 2026                          [오늘]│
├─────────────────────────────────────────┤
│  일  월  화  수  목  금  토              │
├─────────────────────────────────────────┤
│      26  27  28  29  30  31   1         │
│          업무 회의 개인                  │
│          운동     점심                   │ (확장 가능)
│                   공부                   │
│                                         │
│   2   3   4   5   6   7   8            │
│  회의 휴식 개인 업무                     │
│                                         │
└─────────────────────────────────────────┘
```

**선택 기준**:
- **디자인 A**: 구현 간단, 성능 우수, 패턴 파악 쉬움
- **디자인 B**: Timeblocks와 유사, 상세 정보 많음, 구현 복잡

**추천**: 디자인 A (히트맵) → 나중에 B로 업그레이드 가능

### Phase 3: 뷰 전환 및 내비게이션

**탭 구조**:
```typescript
type ViewMode = 'day' | 'week' | 'month'

<ViewModeTabs
  mode={viewMode}
  onModeChange={setViewMode}
/>

{viewMode === 'day' && <PieChart ... />}
{viewMode === 'week' && <WeeklyPieView ... />}
{viewMode === 'month' && <MonthlyHeatmap ... />}
```

**내비게이션 일관성**:
- Day: 좌우 스와이프 → 이전/다음 날
- Week: 좌우 스와이프 → 이전/다음 주
- Month: 좌우 스와이프 → 이전/다음 월

## 📐 구현 우선순위 (Timeblocks 벤치마킹)

### 우선순위 1: 주간 뷰 (2-3일)
**이유**:
- 파이 차트 재사용 → 구현 빠름
- Time Pie 정체성 유지
- 일주일 패턴 파악 (Timeblocks 부족한 부분)

**성공 기준**:
- 7개 미니 파이 차트 렌더링 <100ms
- 스와이프 제스처 부드러움
- 이벤트 개수 정확

### 우선순위 2: 월간 히트맵 (3-4일)
**이유**:
- 장기 계획 가능
- Timeblocks 스타일 차용
- 단순한 히트맵 → 복잡도 낮음

**성공 기준**:
- 30-35 셀 렌더링 <50ms
- 색상 그라데이션 정확
- 클릭 → Day 뷰 전환

### 우선순위 3: 드래그 앤 드롭 (Phase 3, 별도 계획)
**이유**:
- 복잡도 높음
- Framer Motion 활용 필요
- 주간/월간 뷰 완성 후 추가

## 🎯 Time Pie만의 차별화 포인트

### 1. 파이 차트 + 캘린더 하이브리드

**Timeblocks**: 캘린더만
**Time Pie**: 캘린더 + 파이 차트 (시간 분배 시각화)

**장점**:
- 일 단위: 파이 차트 (시간 분배)
- 주 단위: 미니 파이 (패턴)
- 월 단위: 히트맵 (밀도)

### 2. 실행 추적 (Execution Timer)

**Timeblocks**: 계획만 (실행 추적 없음)
**Time Pie**: 계획 + 실행 + 비교

**가치**:
- 계획 vs 실제 비교
- 완료율 추적
- AI 제안 (분석 기반)

### 3. 3단계 이벤트 타입 (Hard/Anchor/Soft)

**Timeblocks**: 단순 이벤트
**Time Pie**: 유연성 레벨 구분

**활용**:
- Anchor: 고정 루틴
- Hard: 중요 약속
- Soft: 유연한 목표

## 📊 최종 추천 아키텍처

```
Time Pie 멀티 뷰 시스템

┌─ Day View (메인) ────────────────────┐
│  - 파이 차트 (24시간 원형)            │
│  - Up Next 리스트                     │
│  - Execution Timer                   │
│  - 시간 분배 시각화 (고유 가치)       │
└──────────────────────────────────────┘
         ↕ (탭 전환)
┌─ Week View (보완) ───────────────────┐
│  - 7개 미니 파이 차트                 │
│  - 일주일 패턴 파악                   │
│  - 이벤트 개수 표시                   │
│  - 스와이프 내비게이션                │
└──────────────────────────────────────┘
         ↕ (탭 전환)
┌─ Month View (장기 계획) ─────────────┐
│  - 이벤트 밀도 히트맵                 │
│  - Timeblocks 스타일 차용             │
│  - 한 달 전체 한눈에                  │
│  - 클릭 → Day 뷰 전환                │
└──────────────────────────────────────┘
```

**핵심 전략**:
1. ✅ **유지**: 파이 차트 (Time Pie 정체성)
2. ➕ **추가**: 주간/월간 뷰 (Timeblocks 벤치마킹)
3. 🔄 **차별화**: 실행 추적 + 시간 분배 분석
